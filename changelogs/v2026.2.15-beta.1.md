# OpenClaw Release 2026.2.15-beta.1

**Released:** February 16, 2026 at 04:12 UTC
**Release Type:** Pre-release
**Commits since release:** 1,290 to main

## Changes

### Discord Enhancements

The release unlocks rich interactive agent prompts through Components v2, enabling "buttons, selects, modals, and attachment-backed file blocks" for native Discord interaction. UI improvements include embeds passthrough and execution approval UX refinements.

### Plugin & Extension Capabilities

Plugins can now observe LLM operations through exposed `llm_input` and `llm_output` hook payloads, allowing extensions to "observe prompt/input context and model output usage details."

### Subagent Nesting

Nested sub-agents (sub-sub-agents) are now supported with configurable depth limits and per-agent child constraints. Configuration allows setting `agents.defaults.subagents.maxSpawnDepth: 2`.

### Channel Customization

Slack, Discord, and Telegram now support "per-channel ack reaction overrides (account/channel-level)" for platform-specific emoji formats.

### Webhook Improvements

Cron/Gateway adds finished-run webhook delivery toggle and dedicated webhook authentication token support via `cron.webhookToken`.

### Channel Type Deduplication

Core and extension probe/token resolution base types are now deduplicated while "preserving per-channel error typing."

## Fixes

**Security:** SHA-1 sandbox hashing replaced with SHA-256; Telegram bot tokens redacted from logs; dangerous Docker configurations blocked; sensitive session details redacted from status responses.

**Sandbox & Config:** Array order preservation in config hashing; restrictive `targetDir` for skill downloads; enhanced go installer fallback on apt systems.

**Agents & Context:** Model context window overrides properly honored; cross-provider cache collisions prevented; explicit timeout error replies for embedded runs.

**Group Chats:** Group context injected into system prompt on every turn, preventing model awareness loss.

**Messaging Platforms:** Telegram message thread handling for DMs; Discord channel session continuity improved; skill command deduplication in multi-agent setups.

**Memory & Search:** Unicode-aware full-text search for non-ASCII queries; timezone-aware memory filename resolution.

**UI/UX:** Searchable-select filtering made ANSI-aware; TUI output formatting preserves copy-sensitive tokens; `BOOTSTRAP.md` hidden post-onboarding.

**Additional:** Auto-reply/streaming improvements; Cron payload kind inference; bracketed paste fallback for Git Bash; browser control service unavailability handling.
