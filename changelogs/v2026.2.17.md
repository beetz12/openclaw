# OpenClaw Release v2026.2.17

**Released:** February 18, 2026 by @steipete
**Commit:** `4134875`
**314 commits to main since this release**

## Changes

### Agents & Models

- Added opt-in 1M context beta support for Anthropic Opus/Sonnet via `params.context1m: true`
- Implemented Anthropic Sonnet 4.6 support across aliases with fallback compatibility
- Enabled `tool_stream` by default for Z.AI with opt-out capability
- Added per-model `thinkingDefault` configuration overrides
- Enabled `llms.txt` discovery in default agent behavior
- Added OpenAI Codex CLI auth provider integration

### Commands & Subagents

- Introduced `/subagents spawn` for deterministic subagent activation
- Enhanced subagent task messages with context preservation
- Improved context-window overflow handling via preemptive truncation
- Added explicit recovery guidance for compacted tool outputs
- Prefixed spawned subagent messages with source information

### iOS Features

- Added iOS share extension forwarding URL/text/image content to gateway
- Implemented "Background Listening" toggle for Talk Mode
- Added "Voice Directive Hint" toggle to reduce token usage
- Hardened barge-in behavior by disabling interrupt-on-speech with built-in speakers
- Restored significant location monitor implementation
- Auto-select local Apple Development team during project generation

### Messaging Platforms

**Slack:**

- Native single-message text streaming via `chat.startStream/appendStream/stopStream`
- Configurable streaming modes for draft previews

**Telegram:**

- Inline button style support (primary|success|danger)
- User message reactions as system events
- DM-topic replies and draft previews preservation
- Emoji reaction action normalization for Telegram menu registration
- IPv4 fallback support via `autoSelectFamily` on Node.js 22+

**iMessage:**

- Support for `replyToId` on outbound text/media sends
- Normalized leading `[[reply_to:<id>]]` tags for proper targeting

**Discord:**

- Native `/exec` command options with autocomplete
- Reusable interactive components (`components.reusable=true`)
- Per-button `allowedUsers` allowlist for authorization
- Optimized reaction notification handling
- String ID requirement in Discord allowlists

**Mattermost:**

- Emoji reaction actions with explicit `remove` flag

### Tools & Web

- URL allowlists for `web_search` and `web_fetch`
- Tool Display/Web UI intent-first detail views and exec summaries
- `read` tool auto-pagination and context-aware output budgeting
- Enhanced tool-loop detection with phased approach and circuit breaker

### Cron & Automation

- Separate per-job webhook delivery mode from announce delivery
- Deterministic default stagger for top-of-hour cron schedules
- Per-run model/provider usage telemetry logging
- Local usage report script for token aggregation
- Session ID reuse for webhook/cron runs with stable keys

### Voice & Media

- Voice Call pre-caches inbound greeting TTS
- Optional stale call reaper (`staleCallReaperSeconds`)
- Auto-end calls when media streams disconnect

### Skills & Configuration

- Compact skill file paths using `~` for home directory
- Refined skill-description routing boundaries
- Added `before_agent_start` model/provider overrides

### Memory & Search

- FTS fallback plus query expansion for memory search
- Scoped managed collection names per agent

### Auto-reply & Metadata

- Include trusted inbound `message_id` in conversation metadata
- Include `sender_id` for moderation workflow targeting
- Preserve `replyToId` across streamed/split message chunks

### Docker & Infrastructure

- Optional `OPENCLAW_INSTALL_BROWSER` build arg for Chromium/Xvfb preinstall
- Session-store atomic writes on Windows

## Fixes

### Agent & Context Issues

- Collapsed image resize diagnostics to single-line logging
- Preemptively guarded accumulated tool-result context
- Stripped duplicated truncation payloads from tool-result details
- Raised default bootstrap prompt cap from 24,000 to 150,000 characters
- Honored `agents.defaults.imageModel` during auto-discovery

### Message & Reply Threading

- Kept reply context sticky across streamed chunks
- Preserved `replyToId` on all chunk sends across platforms
- Prevented stale thread ID leakage into non-thread sessions

### iOS & Client Issues

- Stopped auth Step 3 retry-loop churn with connection pause
- Routed ChatSheet RPCs through operator session for authorization
- Fixed iOS gateway-controller coverage for command surface
- Hardened mobile talk config handling with keychain overrides

### macOS & Updates

- Corrected Sparkle appcast version for 2026.2.15
- Prevented restart crash loops after failed self-updates
- Added standalone restart helper honoring service-name overrides

### Discord & Platform Fixes

- Confined `$include` resolution to top-level config directory

### Gateway & Configuration

- Deferred transient lifecycle error snapshots for `agent.wait`
- Bridged outbound/inbound message lifecycle into internal hook events
- Prevented `config.patch` object-array merges from full replacement
- Trimmed whitespace around trusted proxy entries
- Separated gateway and daemon service-token drift warnings
- Auto-repaired `dmPolicy="open"` configs during doctor fix

### Sessions & Maintenance

- Avoided duplicating typed session prefixes in display names
- Created transcript JSONL files with user-only (0o600) permissions
- Archived transcripts when pruning stale sessions
- Purged `.deleted` transcript archives after prune window

### Hooks & Tool Execution

- Preserved `before_tool_call` wrapped-marker across abort-signal wrapping
- Added preflight guard detecting shell env var injection in scripts
- Treated normal non-zero exit codes as completed
- Delivered tool-result media even with verbose output disabled

### Models & Provider Issues

- Synced auth-profile credentials before registry checks
- Normalized provider-id aliases when bridging credentials
- Probed primary model when auth-profile cooldown nears expiry
- Classified provider abort errors as timeout-class failures

### UI & Display

- Replaced undefined `var(--text-muted)` with `var(--muted)`
- Preserved selected-range totals via bucket-aggregating timeseries

### Process & System

- Disabled detached spawn in exec runs on Windows
- Gracefully terminated process trees with SIGTERM before SIGKILL
- Prevented overlapping closed-loop turn races with per-call locking

### Security & Validation

- Validated base64 image payloads before submission
- Validated catalog entries in `openclaw models set`
- Fixed the OC-09 credential-theft path via environment variables

### CLI Improvements

- Made `/model picker` allowlist prompt searchable
- Preserved `--components` JSON payloads in message sends
- Fixed parent/subcommand option collisions
- Ensured `openclaw doctor --fix` exits promptly
- Preferred `gateway.remote.url` for `openclaw qr --remote`

### Cron & Automation Fixes

- Preserved per-job schedule-error isolation in recompute
- Resolved `accountId` from agent bindings in isolated sessions
- Prevented spin loops when jobs complete within scheduled second
- Preserved default model fallbacks with partial overrides
- Routed text-only announce through main session flow
