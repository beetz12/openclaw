# openclaw Release v2026.2.15

**Released:** February 16, 2026 by [@steipete](https://github.com/steipete)
**Commits since release:** 1,293 to main

---

## Changes

### Discord Enhancements

- Unlocked rich interactive agent prompts using Components v2, introducing buttons, selects, modals, and attachment-backed file blocks for native Discord interaction
- Enhanced UI with Components v2 containers, button layouts, and Discord-forwarding skip capabilities alongside embed passthrough and execution approval refinements

### Plugin Extensions

- Exposed `llm_input` and `llm_output` hook payloads, enabling extensions to observe prompt context and model output usage details

### Subagent Capabilities

- Implemented nested sub-agents (sub-sub-agents) with configurable depth via `agents.defaults.subagents.maxSpawnDepth: 2`
- Added `maxChildrenPerAgent` limit (default 5) with depth-aware tool policies and proper announce chain routing

### Channel Configuration

- Added per-channel acknowledgment reaction overrides at account/channel levels to support platform-specific emoji formats across Slack, Discord, and Telegram

### Cron & Gateway

- Introduced finished-run webhook delivery toggle (`notify`) and dedicated webhook authentication token support for outbound cron webhook posts

### Channel Infrastructure

- Deduplicated probe/token resolution base types across core and extensions while preserving per-channel error typing

---

## Fixes

### Security Improvements

- Replaced deprecated SHA-1 sandbox configuration hashing with SHA-256 for deterministic identity and recreation checks
- Redacted Telegram bot tokens from error messages and stack traces to prevent accidental secret leakage
- Blocked dangerous Docker configurations (bind mounts, host networking, unconfined seccomp/apparmor) preventing container escape
- Sanitized workspace paths before LLM embedding to prevent instruction injection via malicious directory names
- Restricted `download` installer `targetDir` to per-skill tools directory, preventing arbitrary file writes
- Capped response body sizes before HTML parsing to prevent memory exhaustion
- Made sensitive-key whitelist matching case-insensitive while preserving exemptions

### Gateway & Control UI

- Redacted sensitive session/path details from status responses for non-admin clients
- Preserved operator scopes for Control UI bypass modes when device identity is unavailable
- Hardened `chat.send` message handling by rejecting null bytes and normalizing Unicode
- Prevented stored XSS via assistant names/avatars through script injection removal

### Channel-Specific Fixes

- **LINE:** Fail-closed on missing webhook tokens; treat accounts as configured only when both token and secret present
- **Telegram:** Omitted `message_thread_id` for DMs; replaced audio placeholders with transcripts; implemented media retry logic with fallback
- **Discord:** Preserved channel session continuity when payloads omit `channelId`; deduped native skill commands; aligned role matching to raw IDs

### Agent & Model Fixes

- Applied configured model `contextWindow` overrides after provider discovery
- Derived `lookupContextTokens()` from auth-available metadata with smallest context window for duplicates
- Forced `store=true` for direct OpenAI Responses/Codex runs
- Made Unicode-aware FTS queries for non-ASCII and CJK languages
- Always injected group chat context into system prompt on every turn

### Memory & Processing

- Scoped managed collection names per agent and precreated glob-backed directories
- Resolved `memory/YYYY-MM-DD.md` placeholders with timezone-aware dates
- Returned explicit timeout error replies for embedded runs with no payloads
- Inferred `payload.kind="agentTurn"` for partial cron updates

### UI & Terminal Improvements

- Made TUI searchable-select filtering ANSI-aware to preserve styling
- Coalesced rapid multiline pastes in Git Bash when bracketed paste unavailable
- Suppressed false "(no output)" placeholders for concurrent runs
- Preserved copy-sensitive tokens during text wrapping to prevent corruption

### Additional Improvements

- Hardened go installer fallback on apt-based systems
- Hardened git pre-commit hook against option injection
- Made legacy daemon CLI compatibility tolerant of minimal exports
- Enhanced browser unavailability messaging with non-retry guidance
