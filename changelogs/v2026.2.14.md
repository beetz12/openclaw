# openclaw Release v2026.2.14

**Release Date:** February 15, 2026
**Released by:** [@steipete](https://github.com/steipete)
**Commits since release:** 1,997 to main

---

## Changes

### Messaging & Channels

- **Telegram:** Support poll sending via `openclaw message poll` with duration, silent delivery, and anonymity controls
- **Slack/Discord:** Add `dmPolicy` and `allowFrom` config aliases for DM access control; legacy keys remain supported
- **Discord:** Enable exec approval prompts to target channels or both DM+channel via `channels.discord.execApprovals.target`
- **Telegram:** When `channels.telegram.commands.native` is `false`, exclude plugin commands from menu registration
- **LINE:** Accept "Verify" requests from Developers Console without signatures while maintaining requirements for real deliveries
- **BlueBubbles:** Include sender identity in group chat envelopes and align message formatting with iMessage/Signal

### Infrastructure & Sandbox

- **Sandbox:** Add `sandbox.browser.binds` to configure browser-container bind mounts separately
- **Discord:** Add debug logging for message routing decisions to improve `--debug` tracing

### CLI & TUI

- **CLI:** Fix lazy core command registration so maintenance commands resolve correctly
- **CLI/Dashboard:** Generate localhost dashboard URLs when `gateway.bind=lan` to satisfy browser requirements
- **TUI/Gateway:** Resolve local gateway target URL from `gateway.bind` mode instead of hardcoded localhost
- **TUI:** Honor explicit `--session <key>` even when `session.scope` is `global`
- **TUI:** Use available terminal width for session name display
- **TUI:** Preserve in-flight streaming replies when different runs finalize concurrently
- **TUI:** Keep pre-tool streamed text visible when tool-boundary deltas omit earlier blocks
- **TUI:** Sanitize ANSI/control-heavy history and split pathological tokens before rendering

### Agent & Workspace Features

- **Agents:** Deliver tool result media regardless of verbose level
- **Agents/Image tool:** Allow workspace-local image paths and trust sandbox-validated paths
- **Agents:** Harden transcript path resolution for mismatched agent contexts
- **Cron:** Deliver text-only output directly when `delivery.to` is set
- **Cron/Slack:** Preserve agent identity when cron jobs deliver outbound messages
- **Media:** Accept `MEDIA:` prefixed paths when loading outbound media
- **Workspace:** Persist bootstrap onboarding state for partially initialized workspaces
- **Skills:** Watch `SKILL.md` only when refreshing to avoid file-descriptor exhaustion

### Memory & Knowledge Management

- **Memory/QMD:** Make `memory status` read-only by skipping update side effects
- **Memory/Builtin:** Keep `memory status` stable across invocations
- **Memory/QMD:** Cap command output buffering and parse scope keys once per request
- **Memory/QMD:** Use exact docid matches before prefix lookup fallback
- **Memory/QMD:** Avoid multi-collection ranking corruption with per-collection queries
- **Memory/QMD:** Detect and self-heal corrupted collection metadata
- **Memory/Builtin:** Narrow watcher targets to markdown globs and ignore dependency directories
- **Outbound/Memory:** Bound directory cache growth with TTL and max-size pruning
- **Skills/Memory:** Remove disconnected nodes from remote-skills cache

### Security Enhancements

- **Memory/LanceDB:** Treat recalled memories as untrusted; escape injected text and skip injection payloads
- **Memory/LanceDB:** Require explicit `autoCapture: true` opt-in (now disabled by default)
- **Media/Security:** Allow local media reads from workspace and sandboxes roots by default
- **Media/Security:** Harden allowlist bypasses by requiring explicit `readFile` override
- **Discord/Security:** Harden voice message media loading with SSRF and path checks
- **BlueBubbles/Security:** Require explicit `mediaLocalRoots` allowlists for local media
- **BlueBubbles/Security:** Reject ambiguous webhook routing; harden auth behind reverse proxies
- **Feishu/Security:** Harden media URL fetching against SSRF
- **Nostr/Security:** Require loopback source and block cross-origin mutation attempts
- **Signal:** Harden signal-cli archive extraction to prevent path traversal
- **Hooks/Security:** Restrict hook transform modules to `~/.openclaw/hooks/transforms`
- **Archive/Security:** Enforce extraction entry/size limits
- **Media/Security:** Reject oversized base64 input; stream and bound URL-backed fetches
- **Skills/Security:** Harden archive extraction for downloaded skills
- **Slack/Security:** Compute authorization for DM slash commands even when `dmPolicy=open`
- **iMessage/Security:** Keep DM pairing identities out of group allowlist authorization
- **Google Chat/Security:** Deprecate `users/<email>` allowlists; require user IDs
- **Google Chat/Security:** Reject ambiguous shared-path routing
- **Telegram/Security:** Require numeric sender IDs; reject `@username` principals
- **Telegram/Security:** Reject webhook startup without `webhookSecret`
- **Windows/Security:** Avoid shell invocation to prevent metacharacter injection
- **Agents/Security:** Scope CLI cleanup to owned child PIDs; enforce workspace-root bounds for patches
- **Agents/macOS:** Prevent shell injection in keychain credential writes
- **macOS:** Hard-limit unkeyed deep links; require valid unattended keys
- **Gateway/Security:** Harden tool-supplied `gatewayUrl` overrides; block legacy `system.execApprovals`
- **Gateway/Security:** Reject oversized base64 attachments; stop returning raw config values
- **Net/Security:** Fix SSRF bypass via IPv4-mapped IPv6 literals
- **Browser/Security:** Harden file upload/download helpers; block cross-origin mutating requests
- **Exec/Security:** Enforce `system.run` consistency; harden PATH handling
- **Tlon/Security:** Harden Urbit URL fetching; block private hosts by default
- **Voice Call/Security:** Require webhook signature verification for Telnyx and Twilio
- **Discovery/Security:** Prefer resolved endpoints over Bonjour TXT records; require trusted gateways

### Bug Fixes

- **CLI/Plugins:** Ensure `openclaw message send` exits after delivery so one-shot sends don't hang
- **CLI/Plugins:** Run plugin `gateway_stop` hooks before exit for resource cleanup
- **WhatsApp:** Honor per-account `dmPolicy` overrides
- **Cron:** Prevent `cron list`/`cron status` from skipping past-due jobs
- **Cron:** Repair missing/corrupt `nextRunAtMs` during updates
- **Cron:** Skip replay on startup for interrupted jobs
- **Discord:** Prefer guild id when logging inbound messages
- **Discord:** Treat empty channel configs as no allowlist (not deny-all)
- **Models/CLI:** Guard `models status` string trimming to prevent crashes
- **Gateway/Subagents:** Preserve queued announcements on delivery errors
- **Gateway/Sessions:** Abort active runs before reset; clear queued work
- **Agents:** Add timeout around session compaction to ensure stalled runs settle
- **Agents:** Keep unresolved tool failures visible; dedupe warnings
- **Agents/Process:** Preserve pagination; enforce strict bootstrap budgeting
- **Agents:** Classify external timeout aborts same as internal; prevent auth rotation
- **Agents:** Treat empty-stream failures as timeout signals
- **Agents:** Treat `read` tool `file_path` as valid in diagnostics
- **Ollama/Agents:** Avoid forcing `<final>` tag enforcement
- **Plugins:** Suppress false duplicate warnings for extensions in multiple paths
- **Sandbox/Tools:** Make file tools bind-mount aware with read-only semantics
- **Signal:** Preserve case-sensitive `group:` IDs during normalization
- **Telegram:** Set webhook timeout to 10s to prevent retry storms
